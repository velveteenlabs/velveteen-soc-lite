# Findings Guide

This document explains how to interpret findings generated by Velveteen SOC Lite.

---

## Severity Levels

### HIGH

Strong indicators of potentially malicious activity.

Examples:

* Encoded PowerShell commands
* Execution from Temp, AppData, or user-writable directories
* Unsigned or tampered drivers

Action:

* Prioritize investigation immediately
* Validate source and intent
* Correlate with other findings

---

### SUSPICIOUS

Behavior that requires further investigation.

Examples:

* Uncommon persistence mechanisms
* Unexpected network connections
* Services running from unusual paths

Action:

* Investigate context
* Correlate with process and network data
* Determine if behavior is expected

---

### REVIEW

Low-confidence observations.

Examples:

* Use of cmd.exe or PowerShell
* Software updater activity
* Browser helper processes

Action:

* Review only if supported by other findings
* Do not treat as malicious by default

---

## Important Concept

REVIEW ≠ malicious

This toolkit intentionally surfaces benign activity for visibility.

The goal is awareness, not immediate action.

---

## Common False Positives

* Antivirus and security software
* Game launchers and anti-cheat systems
* Browser extensions and helpers
* OEM utilities
* Software update services

---

## What to Validate Before Acting

Before making changes to a system, verify:

1. File path
2. Digital signature
3. Command-line arguments
4. Parent process
5. Network behavior
6. Frequency and repetition

---

## Correlation Matters

A single finding rarely indicates compromise.

Higher confidence comes from:

* Multiple modules flagging the same process
* Repeated suspicious behavior
* Alignment between persistence, execution, and network activity

---

## Analyst Mindset

Focus on:

* Patterns over isolated events
* Context over assumptions
* Behavior over labels

This toolkit is designed to assist human analysis—not replace it.

---

## Final Note

Avoid deleting drivers, services, or files based on a single alert.

Investigate first. Act second.
